# Contributing to Aphelion-rs

## Code of Conduct

Our code of conduct is guided by classical liberal values. The focus is on the project and related contributions. We believe in individual freedom and responsibility. Please be respectful and considerate of others' viewpoints, but feel free to express your ideas and criticisms constructively.

## Getting Started

1. Fork the repository on GitLab.
2. Clone your fork locally.
3. Create a new branch to work on.
4. Make your changes.
5. Push your changes back to your fork on GitLab.

## Merge Requests

1. Create a new merge request from your forked repository to the main project repository.
2. Describe your changes in detail and link any relevant issues.
3. Please ensure your code adheres to Rust's formatting and linting standards (`rustfmt` and `clippy`).
4. Make sure your changes pass all the tests in the CI pipeline.

## Issues

Feel free to submit issues for any bugs, enhancements, or feature requests.
